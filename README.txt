KYOTO DAYS　提出用サイト説明

【基本情報】
1. ホームページ：index.html
2. HTMLページ数：22ページ
3. テーマ：京都観光と留学生生活
4. 使用技術：HTML5 / CSS3 / JavaScript（motion.js / main.js / contact.js / quiz.js）
5. CSS、JavaScript、画像はすべて相対パスで読み込んでいます。
6. 外部CDNや外部JavaScriptライブラリには依存せず、オフラインで閲覧できます。

【デザイン】
・日本の旅行情報誌を参考にした、写真と実用情報中心の編集デザインです。
・明朝体の見出し、墨色と朱色、細い罫線、読みやすい余白を使用しています。
・観光地、交通、食文化の実景写真11点をローカル保存しています。
・実景写真はWebP形式も用意し、画像サイズをHTMLに記載して表示のずれを抑えています。
・写真と防災図の出典・制作情報は「写真・画像出典.txt」に記載しています。

【JavaScript機能】
・ホームと各下層ページで異なる導入モーション
・IntersectionObserverによる一度だけのスクロール表示
・対応ブラウザでのView Transitionsとスクロール連動視差
・動きを止めるボタンとprefers-reduced-motion対応
・スマートフォン用ナビゲーション
・観光スポットのカテゴリ絞り込み
・写真ギャラリーのライトボックス
・京都知識クイズ
・お問い合わせフォームの入力チェック
・FAQアコーディオン
・ページ上部へ戻るボタン

【提出前に行うこと】
・「小组成员分工明细.txt」に実際の氏名と学籍番号を記入してください。
・実際のグループ番号が第41組でない場合は、フォルダ名とZIP名を変更してください。


【デザイン改訂】
トップページは祇園の実景写真と簡潔な案内文で構成し、広告的な大見出しや装飾ラベルを使わない編集型レイアウトに変更しました。動きは粒子や常時ループを使わず、見出し・写真・罫線・一覧の順序が伝わる短い演出に限定しています。全ページで同じ動きの規則を使い、本文の読みやすさを優先しています。

【モーション設計】
・初回表示：ヘッダー、見出し、概要、写真を時間差で表示します。
・本文：見出し、写真、カード、表をグループ単位で一度だけ表示します。
・操作：絞り込み、FAQ、ライトボックス、クイズの状態変化を連続して見せます。
・性能：scrollイベントの連続監視を使わず、transform、opacity、clip-pathを中心に動かします。
・低負荷：タッチ端末・低スレッド端末では視差を省略し、本文表示は維持します。
